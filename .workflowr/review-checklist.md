# Review Checklist

## Correctness

- Acceptance criteria are fully covered.
- Edge cases and error states behave as expected.
- No unrelated behavior changed.

## Tests

- Tests cover the changed behavior.
- Failure paths are tested when relevant.
- Coverage target is met or explained.

## Maintainability

- Implementation follows existing project patterns.
- Names and boundaries are clear.
- New abstractions are justified by real complexity.

## Operations

- Required commands pass.
- No secrets, local paths, or debug artifacts are committed.
- PR description includes testing and risk notes.
